# Filterbank

```@meta
CurrentModule = Blio.Filterbank
```

The `Filterbank` module provides read/write access to SIGPROC Filterbank
files (typically `.fil` files).  A Filterbank file consists of a single header
followed by a contiguous block of spectra.  Each spectrum has `nchans` channels
and `nifs` IFs (polarization products).  Supported bit depths are 8 and 32.


## Headers

[`Filterbank.Header`](@ref) represents a SIGPROC Filterbank header.  It is a
subtype of `AbstractDict{Symbol, Any}` and behaves like an `OrderedDict` with
the added ability to access header items as if they were struct fields.

```jldoctest
julia> using Blio

julia> fbh = Filterbank.Header();

julia> fbh[:source_name] = "Demo"; fbh[:nchans] = 1024;

julia> fbh.source_name
"Demo"

julia> haskey(fbh, :fch1)
false
```

Construct an empty header with the default constructor, copy an existing
header with the copy constructor, or build one from another Dict-like object
or `NamedTuple`:

```julia
fbh = Filterbank.Header(other_fbh)
fbh = Filterbank.Header((source_name = "Demo", nchans = 1024))
```

You can also derive a Filterbank header from a
[`GuppiRaw.Header`](@ref Blio.GuppiRaw.Header):

```julia
fbh = Filterbank.Header(grh)
fbh = Filterbank.Header(grh; nbits = 8, source_name = "MySource")
```

## HDF5 conversion

The `fil2h5` and `h52fil` functions are defined as empty stubs in the
`Filterbank` module so that their docstrings are available even when
[HDF5.jl](https://github.com/JuliaIO/HDF5.jl) is not loaded.  Their actual
implementations are provided by the `HDF5FilterbankExt` extension module.  See
[Extensions](@ref) for details and the API documentation for
[`fil2h5`](@ref Blio.Filterbank.fil2h5) and [`h52fil`](@ref Blio.Filterbank.h52fil).
