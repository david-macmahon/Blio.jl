# GuppiRaw

```@meta
CurrentModule = Blio.GuppiRaw
```

The `GuppiRaw` module provides read/write access to GUPPI Raw files (typically
`.raw` files) produced by several radio telescope backends such as the GUPPI
backend of the Green Bank Telescope (GBT) and the Breakthrough Listen backend of
many telescopes (including the GBT).  A GUPPI Raw file consists of a sequence of
header + data block pairs.  Each header is a series of 80-byte fixed-width
records terminated by an `END` record (similar in spirit to a FITS header).  The
data block that follows is described by the header.


## Headers

[`GuppiRaw.Header`](@ref Blio.GuppiRaw.Header) is the central type used to represent a GUPPI Raw
header.  It is a subtype of `AbstractDict{Symbol, Any}` and behaves much like
an `OrderedDict`, but it also supports dot-syntax for keyword access:

```jldoctest
julia> using Blio

julia> grh = GuppiRaw.Header();

julia> grh[:obsfreq] = 1400.0; grh[:obsnchan] = 64;

julia> grh.obsfreq
1400.0

julia> haskey(grh, :backend)
false
```

Construct an empty header with [`GuppiRaw.Header`](@ref Blio.GuppiRaw.Header) (the default
constructor), or initialize one from an existing `AbstractDict{Symbol, Any}`
or `NamedTuple`:

```julia
grh = GuppiRaw.Header((backend = "GUPPI", obsfreq = 1400.0, obsnchan = 64))
```

## Loading entire files

To read all of the header + data block pairs from a GUPPI Raw file (or set of
files), use [`load`](@ref).  By default it returns a tuple of
`(headers, datablocks)` where each data block is memory-mapped.

```@docs
load
```

## Data block layout

The shape and element type of a GUPPI Raw data block depend on the values of
the header keywords `BLOCSIZE`, `OBSNCHAN`, `NPOL`, `NANTS`, and `NBITS`.  The
helper functions below query those values:

```@docs
blocksize
blocktype
getntime
antnchan
```

### Allocating an array

To allocate a regular (non-memory-mapped) array sized appropriately for a
data block described by `grh`, use [`Base.Array`](@ref Base.Array(::Header, ::Int)):

```@docs
Base.Array(::Header, ::Int)
```

## Coordinate helpers

GUPPI Raw headers sometimes store the right ascension and declination of the
source as strings (e.g. `"HH:MM:SS.S"`) and sometimes as numeric values (in
degrees).  These helpers normalize both forms to a `Float64` in degrees:

```@docs
radeg
decdeg
```
