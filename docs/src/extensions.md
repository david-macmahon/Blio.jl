# Extensions

```@meta
CurrentModule = Blio
```

Blio.jl uses Julia's [extension module](https://docs.julialang.org/en/v1/manual/code-loading/#extension-modules)
mechanism to integrate with several optional downstream packages.  These
extensions are loaded automatically when you `using` the corresponding package
alongside Blio.  None of them are required in order to use the core
`GuppiRaw` and `Filterbank` modules.

## HDF5 — `HDF5FilterbankExt`

Loading [HDF5.jl](https://github.com/JuliaIO/HDF5.jl) alongside Blio enables
the `HDF5FilterbankExt` extension, which provides:

- conversion of SIGPROC Filterbank (`.fil`) files to and from FBH5-style HDF5
  (`.h5`) files via [`Filterbank.fil2h5`](@ref) and [`Filterbank.h52fil`](@ref),
- a [`Filterbank.Header`](@ref Blio.Filterbank.Header) constructor that builds
  a header from an HDF5 `File` or `Dataset`, and
- an `HDF5.write_attribute` method that writes a `Filterbank.Header` as
  attributes of an HDF5 dataset or file.

```julia
using Blio
using HDF5

Filterbank.fil2h5("mydata.fil")            # creates "mydata.h5" that references the data in "mydata.fil"
Filterbank.fil2h5("mydata.fil"; copy=true) # same thing, but copies data into "mydata.h5"
Filterbank.h52fil("mydata.h5")             # creates "mydata.fil" from "mydata.h5" (always copies)
```

## DataFrames — `DataFramesBlioExt`

Loading [DataFrames.jl](https://github.com/JuliaData/DataFrames.jl) alongside
Blio enables the `DataFramesBlioExt` extension.  It provides:

- `DataFrame(::Vector{<:Union{Filterbank.Header,GuppiRaw.Header}})` to build a
  `DataFrame` where each row corresponds to one header and each column
  corresponds to a header keyword (column names are sorted alphabetically),
- `push!(::DataFrame, ::Header)` to append a header as a new row, and
- a `GuppiRaw.Header(::DataFrameRow)` constructor.
- `GuppiRaw.load` methods that accept `::Type{DataFrame}` and return a
  `DataFrame` of headers directly.

```julia
using Blio
using DataFrames

headers, datablocks = GuppiRaw.load("mydata.raw", DataFrame)
df = DataFrame(headers)
```

## DimensionalData — `DimensionalDataFilterbankExt`

Loading [DimensionalData.jl](https://github.com/rafaqz/DimensionalData.jl)
alongside Blio enables the `DimensionalDataFilterbankExt` extension.  It
provides a `dims` method that builds a tuple of `Dim{:Frequency}`,
`Dim{:Pol}`, and `Dim{:Time}` axes suitable for wrapping a Filterbank data
array in a `DimArray`:

```julia
using Blio
using DimensionalData

fbh, data = Filterbank.mmap("mydata.fil")
da = DimArray(data, dims(fbh))
```

The `pols` keyword can be used to label the polarization products:

```julia
dims(fbh; pols = :stokes)   # [:I, :Q, :U, :V]
dims(fbh; pols = :xy)       # [:XX, :YY, :ReXY, :ImXY]
```

| `pols`       | Equivalent `Vector{Symbol}` |
|--------------|------------------------------|
| `:auto`      | `:stokes` if `nifs == 1`, else `:xy` (default) |
| `:stokes`    | `[:I, :Q, :U, :V]`           |
| `:xy`        | `[:XX, :YY, :ReXY, :ImXY]`   |
| `:yx`        | `[:YY, :XX, :ReYX, :ImYX]`   |
| `:lr`        | `[:LL, :RR, :ReLR, :ImLR]`   |
| `:rl`        | `[:RR, :LL, :ReRL, :ImRL]`   |
| `:hv`        | `[:HH, :VV, :ReHV, :ImHV]`   |
| `:vh`        | `[:VV, :HH, :ReVH, :ImVH]`   |
